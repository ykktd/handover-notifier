# Handover Notifier Visual Style

## 目的

このドキュメントは、Handover Notifier の紹介スライド全体の見た目とトーンを定義する。実装時は `design-tokens.md` とあわせて参照する。

## 全体トーン

スライドは、大学サークル向けの清潔で親しみやすいITサービス紹介として作る。

- 友好的
- 現代的
- すっきりしている
- 少しだけ遊びがある
- 企業向けSaaSのように硬すぎない
- 子ども向けのように幼くしない

聞き手はエンジニアとは限らないため、技術の詳細よりも「必要な日に通知が届くと楽になる」という体験を中心に見せる。

## App-Led Visual Direction

デザインの最優先基準は、実アプリのUIとスクリーンショット。

- 淡いベージュ背景に白いカードを置く。
- 深いグリーンを主役にする。
- 通知や年度複製のアクセントとして黄色・オレンジを使う。
- カードは白背景、細い枠線、10px前後の角丸にする。
- 影は控えめにし、必要なUIだけ少し浮かせる。
- Slack / Gmail は実アプリのバッジ色に合わせる。

`asset-policy.md` には「淡いブルーと紫」の記載があるが、これは生成背景の補助色に留める。デッキ全体の主色は、実アプリ由来のベージュ、白、グリーン、黄色とする。

## Slide Composition

基本構成は、短い日本語メッセージと、右側または下部のビジュアルで支える。

- 見出しは1つに絞る。
- 箇条書きは最大3つ。
- 長い説明文を置かない。
- UIスクリーンショットは小さくしすぎない。
- 背景イラストは主役にせず、文字とUIの読みやすさを優先する。
- 表紙とまとめは、余白を広く取り、コアメッセージを強く見せる。

Slide 01 は全体トーンの試作対象、Slide 06 は実UIスクリーンショットの扱いを確認する試作対象とする。

## Text Style

日本語は自然で短く、サークル内で話す言葉に近づける。

使いやすい表現:

- 必要な日に届く
- 思い出しやすい
- 探しに行く負担を減らす
- 前年度の予定をもとにできる
- Webアプリから登録できる

避ける表現:

- リードタイム
- 生産性を底上げ
- 見逃しゼロ
- 完全になくなる
- 自動で予定変更を検知

## Image And Asset Style

GPT Imagesなどの生成画像は、背景、人物、ロボット、装飾に限定する。

### 生成してよいもの

- 通知を受け取る大学生風メンバー
- 資料探しに迷う後輩
- 安心して作業するメンバー
- 小さな通知ボット
- カレンダー、チェック、通知ベルなどの装飾
- 余白のある背景

### 生成しないもの

- 日本語スライド文字
- 詳細な製品説明
- 実アプリUI
- フォーム、ボタン、設定画面
- Slack / Gmail の正確なUI
- Google Drive や Google Calendar の正確なロゴ・画面

人物は聞き手の分身として使い、ロボットはHandover Notifierの補助役として小さめに扱う。人物とロボットが情報を邪魔しないことを優先する。

## Product Accuracy Visual Rules

ビジュアルでも未実装機能を示唆しない。

- Google Calendarの予定表のように見せない。
- LINEなど未実装の通知先を出さない。
- AIが本文を自動生成しているように見せない。
- 既存資料の更新を自動検知しているように見せない。
- 通知ごとにSlackチャンネルを自由に選べるように見せない。
- 通常ユーザーがGoogle Sheetsを直接編集して使うように見せない。
- 複数年度データを並行管理しているように見せない。

Slide 08 の年度更新は「既存通知の日付を+1年し、送信済み状態をリセットする」表現に留める。

## UI Treatment

実UIはスクリーンショットを使う。必要に応じてHTML/CSSで補助的な通知カードやフロー図を作ってよい。

- 通知カードはタイトル、本文プレビュー、日付、Slack/Gmailバッジで構成する。
- フロー図は `Webアプリ -> 登録された通知 -> Apps Script -> Slack / Gmail` の範囲に留める。
- 設定や内部構造の説明は最小限にする。
- 普通の利用者がGoogle Sheetsを触るような見せ方はしない。

## Final Visual Check

- 実アプリと色・角丸・カード感がつながっているか。
- 文字は一目で読めるか。
- 生成画像が主役を奪っていないか。
- スライドごとの役割が混ざっていないか。
- 日本語テキストが画像内に入っていないか。
- 誤った機能を視覚的にも示していないか。
