## 全体トーン

Handover Notifier の実アプリは、淡いベージュ背景に白いカードを置く、やわらかく実用的な1カラム系Webアプリの見た目になっている。雰囲気は「大学サークルの運用ツール」として親しみやすく、企業向けSaaSのような硬さや派手な演出は少ない。

スライドでは、以下の印象を引き継ぐと実アプリとのつながりが出やすい。

- 白いカードと淡い背景による軽さ
- 深いグリーンを主役にした落ち着いた安心感
- 黄色・オレンジ系の通知アクセント
- 丸みのあるUI、やさしい余白、控えめな線
- 日本語が読みやすいシンプルな情報設計

## 色

実アプリの主なCSS変数は以下。

| 用途 | 色 |
|---|---|
| 背景 | `#f0ede8` |
| カード・入力面 | `#ffffff` |
| 境界線 | `#e0dbd2` |
| メインアクセント | `#3d6b52` |
| アクセント淡色 | `#e8f0eb` |
| アクセント中間色 | `#5a8a6e` |
| 本文 | `#1a1a1a` |
| サブテキスト | `#777` |
| ミュート文字 | `#bbb` |
| 危険操作 | `#c0392b` |
| 通知・複製アクセント | `#d08000` |
| 通知・複製背景 | `#fef9ec` |
| Slack | `#4a154b` / `#f3eef5` |
| Gmail | `#c5221f` / `#fdf0ef` |

スライドでは、ベースを白または `#f0ede8` に近い淡色、見出しや主ボタンを `#3d6b52`、通知や年度複製の補助表現を黄色・オレンジ系にするとよい。Slack/Gmail のバッジ色は実アプリに合わせる。

## レイアウト

アプリはモバイルファーストで、中央寄せの1カラム構成を基本にしている。PC幅では通知カードが2カラムになる。

- コンテンツ最大幅は通常 `640px`、大画面では `880px`
- 上部に白い固定トップバー
- 本文は淡い背景の上に中央寄せ
- 一覧では月ごとにセクション分け
- セクション見出しは小さめ太字ラベル + 横罫線 + 件数
- 右下に「通知を追加」の固定FAB

スライドでは、画面全体を広げすぎず、実アプリ由来の「中央にまとまった管理画面」の印象を活かす。実UIスクリーンショットを使う場合は、白いカードと淡い背景が見える余白を残す。

## カード

通知カード、フォームカード、設定パネルはいずれも白背景・薄い枠線・小さめ角丸で統一されている。

- 通知カード: `border-radius: 10px`、`padding: 14px 16px`
- メッセージ入力カード: `border-radius: 12px`、`padding: 16px`
- 設定パネル: `border-radius: 10px`、`padding: 16px`
- モーダル: `border-radius: 14px`、やや強めの影

カードの影は通常ほとんど使われず、枠線で区切るデザイン。FABとモーダルには影がある。スライドでもカードを多用しすぎず、実UIに合わせて「白面 + 細い枠線 + 必要最小限の影」にする。

## ボタン

主ボタンは深いグリーンの塗り、白文字、太字。角丸は8px前後。

- 主ボタン: `#3d6b52` 背景、白文字、`border-radius: 8px`
- セカンダリ: 白背景、薄い枠線、グレー文字
- 危険操作: 薄い赤背景、赤文字、赤系枠線
- テスト送信: 薄い黄色背景、黄色文字、黄色系枠線
- FAB: 深いグリーン、丸みの強いピル形、影あり

スライド上のCTA風パーツや強調ラベルは、主ボタンの見た目を参考にする。黄色のボタンは「通知」「複製」「注意を引く補助」に限定すると実アプリらしい。

## フォーム

追加・編集画面では、通知メッセージ欄が最も大きく扱われている。

- 通知メッセージは大きなテキストエリア
- テキストエリアはアクセントグリーンのやや太い枠線
- 文字数カウンターあり
- タイトル、送信日、参考リンクは通常の白い入力欄
- 送信先は Slack / Gmail のセグメントボタン
- 補足文は薄いグレーで短く表示

Slide 06 では、この「メッセージを主役にするフォーム」を大きく見せるのがよい。架空の入力項目を足さず、実UIにある項目だけを説明する。

## アイコンと装飾

アプリ内アイコンはインラインSVGで、線画中心のシンプルなもの。

- ロゴは深いグリーンの角丸正方形に白いベル、黄色の時計アクセント
- 一覧アクションは紙飛行機、編集、削除
- 設定は歯車
- 送信先は Slack / Gmail アイコン
- 年度複製にはカレンダー/コピー系のアイコン

装飾は控えめで、アイコンは機能理解の補助として使われている。スライドでも、ベル、紙飛行機、カレンダー、チェック、Slack/Gmailバッジなどを小さく使うとアプリらしさが出る。

## フォントと文字感

`Noto Sans JP` を使用している。ウェイトは `400`、`500`、`700` が中心。

- 通常UIは14px前後
- カードタイトルや画面タイトルは太字
- 補足や件数は小さめでグレー
- 行間はやや広めで読みやすい

スライドでは、見出しは大きくしてよいが、文字の雰囲気は太すぎないNoto Sans JP系に合わせる。日本語テキストは短く、余白を広めに取る。

## スライドへ引き継ぐデザイン方針

- 背景は白または淡いベージュを基本にする。
- 主色は深いグリーン、補助色は黄色・オレンジにする。
- Slack/Gmail は実アプリのバッジ色に合わせる。
- UIカードは白背景、細い枠線、10px前後の角丸にする。
- 大きな装飾より、余白・カード・バッジ・小アイコンで見せる。
- 生成画像を使う場合も、実UIより目立たせすぎない。
- スライドの日本語本文はHTML/CSSテキストとして配置し、画像内には入れない。
