# Slide 03: 課題の本質

## Role

抜け漏れは個人の努力不足ではなく、仕組みで改善できる問題だと示す。

## Audience reaction

「自分が悪い」ではなく、確認し続ける運用そのものに無理があると受け止めてもらう。

## Display text

```text
抜け漏れは、
努力不足ではなく仕組みの問題。

・忙しいと確認を忘れる
・資料の場所を思い出せない
・後輩が常に気にし続ける
```

## Visual direction

- 予定、資料、チャット、タスクカードに囲まれて焦る学生
- 忘れていたタスクを示すアイコン
- 重すぎないが、課題感が伝わる雰囲気

## Required assets

- 背景と人物はGPT Images
- 文字はHTML/CSS

## Avoid

- 聞き手を責める表現にしない
- 過度に暗い、重すぎる雰囲気にしない
- 未実装の自動検知機能を示唆しない

## Implementation notes

- 「仕組みの問題」を強く読ませる
- 忙しさや心理的負担は人物表情と周辺カードで表現する
