# Slide 08: 年度交代への対応

## Role

このツールが単発ではなく、毎年の役職交代で使えることを示す。

## Audience reaction

一度作った通知予定を、翌年の引き継ぎにも活かせそうだと感じてもらう。

## Display text

```text
来年の引き継ぎにも、
そのまま使える。

・前年度の予定をベースにできる
・通知日を次年度に更新
・送信済み状態をリセット
```

## Visual direction

- 2025年度の通知一覧から2026年度の通知一覧へコピーされる図
- 通知日が1年進む表現
- 小さなロボットが年度更新を手伝う

## Required assets

- 年度比較と通知カードはHTML/CSS
- ロボットはGPT Images

## Avoid

- 実装にない機能を足さない
- 複数年度のデータを自動で並行管理するように見せない
- 通知内容まで自動生成されるように見せない

## Implementation notes

- 前年度予定をベースにできること、通知日を次年度に更新すること、送信済み状態をリセットすることを正確に示す
- ロボットは年度更新を手伝う補助役として小さめに扱う
